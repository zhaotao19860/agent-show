# Changelog

All notable changes to Agent Show are documented here.
The format is loosely based on [Keep a Changelog](https://keepachangelog.com/),
and this project adheres to [Semantic Versioning](https://semver.org/).

## [1.11.0] — Agent Show migration, Comate, and hardening

### Added
- Comate adapter support for local sessions from `~/.comate-engine`.
- Additional skill roots for Codex and Comate skill directories.
- Embedded web build step so `cargo install --path .` builds the bundled UI.

### Changed
- User-facing app, documentation, and UI branding now use Agent Show / `agent-show`.
- Analytics daily trend now uses last activity time, returns a complete date range, and renders as a clearer SVG trend chart.

### Security
- Local server refuses non-loopback binds unless `--unsafe-public` is explicitly set.
- Mutating HTTP APIs validate local `Origin` / `Referer` headers to reduce CSRF risk.
- Project-level skill install/uninstall and open-directory actions are restricted to known local projects.
- Build-time dependency install uses `npm ci --ignore-scripts` to reduce supply-chain script risk.

## [1.0.0] — Public release

First public release. API stabilization and full documentation pass.

### Stabilized
- `agent-show-core` `AgentAdapter` trait — adapters can now rely on a stable
  contract going forward.
- REST + WebSocket gateway shape (`/api/sessions`, `/api/skills`,
  `/api/prompts`, `/api/conversation`, `/ws`).
- CLI flags: `agent-show serve --bind --no-open`.

### Documentation
- New hero-style README (English + 简体中文).
- Fresh screenshot tour generated by the new Playwright capture suite
  (`e2e/screenshots.spec.ts`).
- Roadmap section reflects v0.1–v0.9 as shipped, v1.0+ as forthcoming.

## [0.9.0] — Digest, cluster, compare

- **Daily / weekly digest export.** One-click Markdown summary covering activity,
  cost, hot files, dangerous tool calls, top tools/skills, peak hours.
- **Prompt clustering.** Toggle-on grouping of similar prompts (Jaccard ≥ 0.45).
- **Side-by-side session compare.** Shift+click two sessions in the sidebar →
  diff view of stats, tools, skills, and prompt overlap.

## [0.8.0] — Token & cost tracking

- Per-turn token chips on every assistant message (↓in / ↑out / cache / $).
- Session-level rollup with by-model breakdown.
- Pricing table covering Opus 3.x–4.7, Sonnet 3.5–4.6, Haiku, GPT-5,
  GPT-5-Codex, GPT-4.1, GPT-4o.
- New `agent-show-core` types: `TurnUsage`, `TokenSummary`.

## [0.7.0] — Skills taxonomy + project discovery

- Built-in taxonomy with 244 entries; skills auto-classified by category.
- Auto-discovery of project-local skill libraries:
  `.github/skills/`, `.agents/skills/`.
- Skill content viewer endpoints.
- Cross-agent conversation integration tests for the MultiAdapter.
- Conversation flow now works for Claude and Codex (was Copilot-only in v0.6).

## [0.6.0] — Conversation flow

- New ConversationFlow tab: agent ↔ LLM ↔ tool unified timeline.
- WebSocket realtime push with version-based dedup.
- Native virtualization via `content-visibility: auto`.
- Polish: deep links, autoscroll, secret highlighting, copy buttons.

## [0.5.x] — Cost analytics + power features

Highlights across v0.5.0 → v0.5.4:
- Cost estimation, model pricing, today/week/month toggles, 7-day sparkline,
  forecast, budget markers.
- Heartbeat heatmap, dangerous tools, hot files with prompt samples.
- Tool inspector popover, opinionated Insights digest card, LivePin floating
  widget for active sessions, Cmd+E export shortcut.
- Localized ErrorBoundary safety net.

## [0.4.x] — Skills + prompts + drilldown

- Skills coverage view, full-text prompt search, tool-call drilldown, star+tag
  filtering. UI polish pass.

## [0.3.x] — Codex adapter

- Codex CLI adapter reading `~/.codex/state_*.sqlite` plus rollout files.

## [0.2.x] — Multi-adapter

- Claude Code adapter, multi-adapter fan-out, activity heatmap.

## [0.1.x] — Initial Copilot CLI dashboard

- Copilot CLI session-state reader, real-time updates, embedded React UI,
  single-binary distribution.
