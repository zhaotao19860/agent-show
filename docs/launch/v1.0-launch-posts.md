## 2. Reddit — r/rust

**Title:** `Agent Show 1.0: a local web dashboard for inspecting Claude Code / Copilot CLI / Codex CLI sessions, written in Rust`

**Body:**

> Hey r/rust! I just shipped Agent Show 1.0 and figured this crowd would appreciate the stack.
>
> **What it is:** A read-only, local-only web dashboard that watches your `~/.claude`, `~/.copilot`, and `~/.codex` directories and shows every session — prompts, tool calls, tokens, cost, errors — in real time. Single binary, no cloud, no telemetry.
>
> **Why I built it:** I'm constantly running multiple CLI agents in parallel (Claude Code in one terminal, Copilot CLI in another, sometimes Codex too). Every agent stores its session as JSONL on disk, but the format is different and there's no way to see "what is each of them doing right now" at a glance. Agent Show is that view.
>
> **The Rust side:**
> - `axum` for the HTTP/WebSocket gateway
> - Per-adapter parsers (`agent-show-claude`, `agent-show-copilot`, `agent-show-codex`) implementing a single `Adapter` trait
> - Each adapter streams its agent's JSONL files; no full-file slurps (the Codex rollouts get to 240k+ lines)
> - File watcher → broadcast channel → WS push → React hooks rerender
> - Workspace versioning, ~30 crates, all `cargo check` clean
> - Conversation flow view rebuilds turns from raw events client-side — same shape regardless of agent
>
> **The dashboard side:** React 19 + Vite + Tailwind v4. Shipped as static assets baked into the Rust binary via `rust-embed`. So one `agent-show serve` starts everything.
>
> **Install from source:**
>
> ```bash
> git clone https://github.com/zhaotao19860/agent-show.git
> cd agent-show
> cargo install --path .
> ```
>
> Repo (MIT): https://github.com/zhaotao19860/agent-show
>
> Happy to talk about any of the parser internals, the conversation reconstruction, or the embed pipeline. Feedback / bug reports very welcome — v1.x is going to be driven by what hurts first for actual users.

## 3. Hacker News — Show HN

**Title:** `Show HN: Agent Show – Local dashboard for Claude Code, Copilot CLI, Codex CLI sessions`

**URL:** `https://github.com/zhaotao19860/agent-show`

**Text (optional but recommended for Show HN):**

> Agent Show is a local-only, read-only web dashboard for inspecting CLI coding agent sessions while you use them. It supports Claude Code, GitHub Copilot CLI, and Codex CLI today; the adapter layer is a trait so adding more is straightforward.
>
> Why I made it: with three CLI agents constantly running it became hard to remember what each one was actually doing — what prompts I'd sent, what tools they'd called, how much each turn cost, whether something silently errored 20 minutes ago. The information is all on disk in `~/.claude`, `~/.copilot`, `~/.codex`, but in three different formats. Agent Show reads those files (never writes), unifies them into a single Sessions list, and adds a few views I kept wanting:
>
> - Overview: every agent's last N sessions, side by side
> - Conversation flow: prompts → assistant turns → tool calls → results, reconstructed from raw events
> - Tools tab: every tool call with args, latency, exit status, full output
> - Insights: token / cost / error trends across all agents over time
> - Skills: which Claude/Copilot skills got loaded into context, ranked
>
> Stack: Rust (axum, tokio, rust-embed) + React 19 + Tailwind v4. Ships as a single binary; `agent-show serve` starts the gateway and serves the bundled dashboard at 127.0.0.1:7777. Read-only filesystem access, no network egress.
>
> v1.0 today, MIT-licensed. The roadmap (v0.1 → v1.0) and architecture are in the README.
>
> Source build instructions are in the README; install with `cargo install --path .`.
>
> Feedback hugely appreciated, especially from people running multiple agents in parallel.
